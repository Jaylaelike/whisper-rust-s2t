🎤 ThaiPBS Transcription API

Upload audio files and get speech-to-text transcription results with optional risk analysis

📋 API Endpoints (Queue System)

POST /api/transcribe

Upload an audio file and queue it for transcription with optional risk analysis

Parameters:
  • audio - Audio file (multipart/form-data)
  • language - Language code (query parameter, default: "th")
  • backend - Processing backend: "cpu", "gpu", "coreml", "auto" (query parameter, default: "auto")
  • chunking - Enable chunking for long audio (query parameter, default: "true")
  • risk_analysis - Enable content risk detection (query parameter, default: "false")
  • priority - Queue priority (query parameter, higher = processed first)
Response: Returns task ID for tracking progress
POST /api/risk-analysis

Queue text content for risk detection analysis

Body:
  • {"text": "content to analyze", "priority": 1} - JSON with text and optional priority
Response: Returns task ID for tracking progress
GET /api/task/{id}/status

Get the status and result of a queued task

Response: Task status, progress, and result (if completed)
GET /api/queue/stats

Get current queue statistics

Response: Pending, processing, completed, and failed task counts
GET /api/queue/history

Get task history with optional filtering

Parameters:
  • limit - Maximum number of tasks to return
  • status - Filter by status: pending, processing, completed, failed
GET /api/health

Check API health status with queue statistics

GET /api/languages

Get list of supported languages

GET /ws

WebSocket endpoint for real-time queue updates and task progress

Events: new_task, task_progress, task_completed, task_status_update

🎵 Queue Audio for Transcription

Backend Notes:
• Auto mode intelligently selects the best available backend
• CPU mode is most reliable and recommended for production use
• GPU/CoreML modes may show harmless "ggml_metal_free" messages in logs
• System automatically falls back to CPU if GPU/CoreML fails
Higher priority tasks are processed first. Use normal priority unless urgent.
⚠️ Risk Analysis Notice:
• Requires LlamaEdge server running on port 8080
• Analyzes transcribed text for harmful content
• Processing will continue even if risk analysis fails
• Check task status for risk analysis results

📝 Example cURL Commands (Queue System)

Queue Audio Transcription (Thai):

curl -X POST "http://localhost:8000/api/transcribe?language=th&backend=auto&priority=1" \
  -F "audio=@your_audio.mp3"

Queue Transcription with Risk Analysis:

curl -X POST "http://localhost:8000/api/transcribe?language=th&backend=auto&risk_analysis=true" \
  -F "audio=@your_audio.mp3"

Check Task Status:

# Replace TASK_ID with the ID returned from transcribe/risk-analysis
curl "http://localhost:8000/api/task/TASK_ID/status"

Queue Risk Analysis:

curl -X POST "http://localhost:8000/api/risk-analysis" \
  -H "Content-Type: application/json" \
  -d '{"text":"มาเล่นบาคาร่าออนไลน์กันเถอะ ได้เงินง่ายๆ", "priority": 1}'

Get Queue Statistics:

curl "http://localhost:8000/api/queue/stats"

Get Task History:

# Get last 10 completed tasks
curl "http://localhost:8000/api/queue/history?limit=10&status=completed"

Health Check:

curl "http://localhost:8000/api/health"

Get Supported Languages:

curl "http://localhost:8000/api/languages"
